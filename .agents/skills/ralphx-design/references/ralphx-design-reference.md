## Design Standard

- Match the existing product surface before adding new visual language.
- Prefer dense, calm, operational UI for RalphX.
- Use the warm-orange accent intentionally.
- Avoid purple/blue-dominant palettes.
- Use existing icons, controls, tokens, and component patterns.
- Include loading, empty, error, disabled, overflow, and responsive states when relevant.
- Verify with focused tests, browser checks, screenshots, or explicit manual inspection notes.
