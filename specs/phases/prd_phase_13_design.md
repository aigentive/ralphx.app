# RalphX - Phase 13: Design System & Requirements

## Overview

This phase transforms RalphX from "ugly and unpolished" to a premium, beautiful Mac app. It establishes a proper design system using Lucide icons and shadcn/ui, then documents specific design requirements for every page/view in the application.

**Reference:** See `specs/DESIGN_OVERHAUL_PLAN.md` for the complete design overhaul strategy.

## Goals

1. Install and configure Lucide React for consistent iconography
2. Install and configure shadcn/ui for premium component primitives
3. Create `specs/DESIGN.md` - the master design document
4. Document design requirements for every page (Phase 12 components included)
5. Define acceptance_criteria and design_quality arrays for Phase 14 implementation

## Dependencies

- Phase 12 must be complete (UI gap components exist)
- All views and components from Phases 5-12 should be functional

## Key Decisions

- **Icons:** Lucide React (replacing all inline SVGs)
- **Components:** Full shadcn/ui adoption (migrate all 57+ components)
- **Design Document:** `specs/DESIGN.md` as single source of truth
- **Skill:** Use `/frontend-design` skill for design planning
- **Verification:** Screenshots via agent-browser for visual verification

## Anti-AI-Slop Guardrails (CRITICAL)

All design work MUST avoid:
- ❌ Purple/blue gradients
- ❌ Inter font (use SF Pro)
- ❌ Generic icon grids
- ❌ High saturation on dark backgrounds
- ❌ Flat, lifeless surfaces

All design work MUST include:
- ✅ Warm orange accent (#ff6b35)
- ✅ Layered shadows for depth
- ✅ Micro-interactions (hover lift, press scale)
- ✅ Typography refinement (letter-spacing, line-height)
- ✅ Strategic accent usage (5% rule)
