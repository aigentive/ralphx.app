# Hook Event Visibility in Chat UI
## Problem
Hook events (SessionStart, PostToolUse, Stop) are currently invisible in the chat feed. Users cannot see when hooks fire, what output they produce, or when hooks block agent progress.
## Solution
Parse hook-related system messages from the Claude stream and render them as lightweight, inline UI elements in the chat feed. The design should be unobtrusive by default with expand-to-inspect capability.
## Architecture
StreamProcessor receives raw text lines from the Claude process stdout. We need to detect hook lifecycle markers within the system message stream and emit typed StreamEvent variants that the frontend can render.
