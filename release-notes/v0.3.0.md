# RalphX.app v0.3.0

RalphX.app 0.3.0 makes Agents the default project workspace, adds isolated agent workspaces with built-in publishing, and reorganizes persistent conversations and artifacts around project-scoped workflows.

## Highlights
- **Agents is now the default project workspace.** RalphX now opens into a dedicated Agents view with a starter composer for `Chat`, `Agent`, and `Ideation`, keeping conversation, runtime, workspace, branch, and relevant artifacts in one project-scoped surface. [3eca21ee](https://github.com/aigentive/ralphx.app/commit/3eca21eeaa2d44a8eae5499f35a0f831484ccfcd)
- **Conversations are now persistent and manageable.** Titles are editable; archive/restore, archived counts, sidebar pagination, search, and creation-time ordering are built in; and sidebar selection, runtime, artifact state, and focused task persist across remounts and mode switches, with child-run and verification chats kept visibly attached to the parent workflow. [3eca21ee](https://github.com/aigentive/ralphx.app/commit/3eca21eeaa2d44a8eae5499f35a0f831484ccfcd)
- **Branch-backed workspaces are visible and actionable.** `Agent` and `Ideation` conversations can use isolated conversation workspaces that surface branch/base branch, freshness, commit history, changed-file counts, and PR status, and eligible workspaces now get an integrated terminal drawer with open, restart, clear, resize, docking, and preserved output. [3eca21ee](https://github.com/aigentive/ralphx.app/commit/3eca21eeaa2d44a8eae5499f35a0f831484ccfcd)
- **Publishing and workspace repair now live inside Agents.** The new `Commit & Publish` flow shows workspace metadata, diff review, publish progress, durable publish history, and clearer error details, and adds freshness checks, update-from-base actions, PR-aware actions, and repair flows that route failures back into the owning conversation and auto-publish repaired workspaces on success. [3eca21ee](https://github.com/aigentive/ralphx.app/commit/3eca21eeaa2d44a8eae5499f35a0f831484ccfcd)
- **Artifacts and task details now sit beside the conversation.** Agents gains an artifact pane for Plans, Verification, Proposals, Tasks, Graph/Kanban views, child transcripts, and orchestration widgets, with ideation-specific tabs shown only when relevant; task details now cover execution, review, merge, merged, waiting, escalated, and incomplete states, and verification history/lineage is easier to inspect. [3eca21ee](https://github.com/aigentive/ralphx.app/commit/3eca21eeaa2d44a8eae5499f35a0f831484ccfcd)

## Fixes And Polish
- **Chat composition and streaming are more workspace-aware.** The composer was expanded, attachments and mode actions moved into a cleaner action menu, runtime info now stays correct per chat context, and the message view improves live streaming, scroll anchoring, bottom-scroll behavior, timestamps, markdown/code styling, copy actions, and message sizing; local file links now open through the system opener. [3eca21ee](https://github.com/aigentive/ralphx.app/commit/3eca21eeaa2d44a8eae5499f35a0f831484ccfcd)
- **Plans, proposals, reviews, and merged-task details render more cleanly.** Plans and proposals now use markdown rendering with better spacing, medium-length review bodies expand inline with modal fallback only for very large reviews, and merged-task metrics, commit messages, markdown lists, and inline code received tighter presentation. [3eca21ee](https://github.com/aigentive/ralphx.app/commit/3eca21eeaa2d44a8eae5499f35a0f831484ccfcd)
- **Welcome, navigation, and theming were refreshed.** The welcome navbar was simplified, `Cmd/Ctrl+N` now creates a project from welcome, the app now defaults to dark theme, theme switching between Dark, Light, and High Contrast is faster, and light/high-contrast styling was improved across Agents, chat, terminal, sidebar, artifact panes, and task details. [3eca21ee](https://github.com/aigentive/ralphx.app/commit/3eca21eeaa2d44a8eae5499f35a0f831484ccfcd)
- **Terminal and artifact panels open more responsively.** Heavy hydration now waits until after the shell paints, which keeps the Agents workspace usable while opening those side surfaces. [3eca21ee](https://github.com/aigentive/ralphx.app/commit/3eca21eeaa2d44a8eae5499f35a0f831484ccfcd)

## Other
- **Workspace recovery is stricter and more reliable.** Startup/shutdown, queue resume, PR polling, publish repair, and ideation launched from Agents now understand conversation workspaces more consistently, and workspace-backed runs no longer silently fall back to the main checkout when the expected workspace is missing or unsafe. [3eca21ee](https://github.com/aigentive/ralphx.app/commit/3eca21eeaa2d44a8eae5499f35a0f831484ccfcd)
