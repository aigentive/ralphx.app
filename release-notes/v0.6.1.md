# RalphX.app v0.6.1

RalphX.app 0.6.1 fixes installed-app agent tool discovery, aligns embedded terminal PATH behavior, and updates Codecov coverage gate handling.

## User-Facing Changes

- Installed macOS app agent runs now hydrate subprocess `PATH` for both Claude and Codex harnesses, so shell commands can find common development tools such as `rustc`, `cargo`, `rg`, Homebrew tools, and shims from rustup/cargo, asdf, pyenv, nodenv, or Volta. [6bb52326](https://github.com/aigentive/ralphx.app/commit/6bb523265a7e88630606a89c58db264f4a33eb22)
- The embedded terminal now reuses the same PATH behavior as agent subprocesses, keeping command discovery consistent when RalphX.app is launched from Finder or another environment with a stripped PATH. [6bb52326](https://github.com/aigentive/ralphx.app/commit/6bb523265a7e88630606a89c58db264f4a33eb22)

## Developer And Maintainer Changes

- The custom Coverage Gate wrapper now accepts Codecov’s `codecov/patch` result from GitHub check runs as well as the legacy commit-status API, avoiding false timeouts when patch coverage is reported through checks. [6bb52326](https://github.com/aigentive/ralphx.app/commit/6bb523265a7e88630606a89c58db264f4a33eb22)
- Added focused regressions for the shared PATH helper, Claude spawn environment, Codex spawn environment, and embedded terminal PATH behavior. [6bb52326](https://github.com/aigentive/ralphx.app/commit/6bb523265a7e88630606a89c58db264f4a33eb22)

## Known Issues

- PATH hydration is additive; tools installed only in uncommon custom directories may still require those directories to be present in the inherited PATH or a supported shim location. [6bb52326](https://github.com/aigentive/ralphx.app/commit/6bb523265a7e88630606a89c58db264f4a33eb22)
