# RalphX.app v0.6.0

RalphX.app 0.6.0 improves installed-app GitHub authentication, reworks Settings startup behavior, and strengthens install documentation plus coverage/release gates.

## User-Facing Changes
- GitHub CLI authentication can now be repaired from inside RalphX for Finder/Homebrew launches: Settings and Agents publish surfaces expose `Sign in` and `Use Terminal` actions, including the one-time GitHub code prompt when `gh` provides one. [4c20b8a1](https://github.com/aigentive/ralphx.app/commit/4c20b8a1e85f636bd6264802465b9ae4837e057e)
- Settings now opens workspace-first with Repository as the default section, remembers the last selected section globally across projects, and paints the dialog shell/close state before lazy-loading heavier section content. [103530b6](https://github.com/aigentive/ralphx.app/commit/103530b68962bec33343a1d1501ffd7a282712b5)
- Install guidance is easier to navigate: the README keeps the Homebrew happy path up front, while Homebrew upgrade, repair, uninstall, data-removal, build-from-source, and coverage details now live in dedicated docs. [022ae071](https://github.com/aigentive/ralphx.app/commit/022ae0715c136ecbf0bdfc00eaf915eda09c2fe3)

## Fixes And Polish
- Git readiness messaging is more precise for SSH repositories: SSH fetch/push health is kept separate from GitHub CLI PR API access, so missing `gh` auth no longer appears as a generic Git warning for all-SSH repos. [4c20b8a1](https://github.com/aigentive/ralphx.app/commit/4c20b8a1e85f636bd6264802465b9ae4837e057e)

## Developer And Maintainer Changes
- Coverage reporting now runs separately from fast CI and publishes Rust, frontend, and plugin reports to Codecov with suite flags for `rust-lib`, `rust-ipc`, `frontend`, `plugin-internal-mcp`, and `plugin-external-mcp`; coverage badges and suite docs were reorganized for clearer public/contributor visibility. [96640624](https://github.com/aigentive/ralphx.app/commit/9664062404f5572b677a4971b67bac0d15cfc743) [022ae071](https://github.com/aigentive/ralphx.app/commit/022ae0715c136ecbf0bdfc00eaf915eda09c2fe3)
- Pull request coverage enforcement now uses a 90% aggregate patch-coverage target plus an always-running Coverage Gate job that can be required in branch protection without blocking workflow-only or dependency-manifest-only PRs that intentionally skip expensive coverage jobs. [bfc09136](https://github.com/aigentive/ralphx.app/commit/bfc09136eec547173b0c4e909a18bc4fbf65dce2) [564158be](https://github.com/aigentive/ralphx.app/commit/564158be119e49ee28a44f62ab5c30ff01f635e8) [bb8b8e94](https://github.com/aigentive/ralphx.app/commit/bb8b8e94a3ac260fb8dbaf03ca1b2c7ca75f102a) [cdaa0f47](https://github.com/aigentive/ralphx.app/commit/cdaa0f47960c7b14a9f60915310f38131627c293)
- Tauri runtime/tooling alignment was upgraded for the 2.11 package set, including `@tauri-apps/api`, `@tauri-apps/cli`, `tauri`, and `tauri-build`, with a CI alignment check to catch future JS/Rust drift. [3db1c9eb](https://github.com/aigentive/ralphx.app/commit/3db1c9eb847ec2ef35287139cfee7be156e6642a)
- CI and test infrastructure were refreshed with newer GitHub Actions, plugin Vitest updates, a stabilized delegated transcript refetch regression test, and visual snapshots for every Settings section.
