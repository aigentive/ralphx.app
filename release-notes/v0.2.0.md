# RalphX.app v0.2.0

RalphX.app 0.2.0 expands theming to Dark, Light, and High Contrast, stabilizes PR-mode and verification flows, and improves planning and chat surfaces.

## Highlights
- Dark, Light, and High Contrast themes now cover the main app surfaces, with Settings controls for theme, motion, and font scale plus color-safe status/link treatment. Theme selection is persisted and applied before hydration to avoid startup flashes. `f6859005`
- PR mode stays aligned with GitHub more reliably: draft PRs are created only after reviewable changes exist, requested-changes states are handled explicitly, and startup recovery backfills missing drafts while keeping plan branches fresh after local merges. `484240bb`, `99e2ffe0`
- Verification runs are easier to follow: the run picker copy and lineage disclosure were reworked, cleanup widgets now prefer terminal state over stale intermediate status, and filesystem MCP actions render through richer chat widgets instead of raw payloads. `484240bb`
- Planning and merge review surfaces now handle missing state more cleanly and show more context. The graph renders a "No plan selected" state instead of crashing, and plan-merge tasks show clearer review context with faster merge polling and a cleaner context rail. `c100b9d5`, `3c2c55e8`

## Fixes And Polish
- Floating chat and reviews panels now size and stack more predictably across views: panels without the execution bar lose the empty bottom strip, and opaque wrappers replace transform-driven slide-ins that let underlying controls bleed through. `f6859005`
- Theme switching no longer fights the user: choosing Dark persists across reloads, and moving between Dark and High Contrast no longer bounces back because of stale switch echoes or mismatched theme state. `f6859005`
- High Contrast is more reliable in Tauri/WKWebView: literal canvas and dialog colors prevent transparent backdrops or blank surfaces, and Select triggers keep visible borders instead of collapsing to label-only controls. `f6859005`
- Reviews, Kanban, and Plan Browser now share the same empty-state primitive, keeping icon treatment, spacing, and action placement consistent with the new Graph no-plan/no-results states. `f6859005`, `c100b9d5`
- External MCP startup now survives symlinked plugin installs by resolving the real entry path, avoiding silent exits followed by repeated failed health probes. `e3b49f1c`

## Other
- macOS release packaging and publishing were reworked for RalphX.app, with split build/publish workflows, fast-fail signing checks, traced build stages, and a publish-only recovery path for reruns. `53b7216a`, `8cc378d4`, `ae8d3824`, `6779ee40`, `4f2bc378`
