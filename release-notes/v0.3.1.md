# RalphX.app v0.3.1

RalphX.app 0.3.1 focuses on production bundle startup reliability and more traceable release-note automation.

## Highlights
- Production bundle startup is stabilized by enforcing strict production chunk execution order, preventing dependency chunks from calling app-entry re-exports before initialization; the rebuilt release binary was verified to render the UI instead of blanking, and a simple Codex project-chat smoke completed. [8d569e24](https://github.com/aigentive/ralphx.app/commit/8d569e240907c792a95665c2b993fc984eacbb77)
- Release-note generation now preserves clearer traceability by enriching squash-style commits such as `(#72)` with associated GitHub PR title/body/URL when `gh` is available, and by requiring exact clickable commit links in generated proposals and notes. [055e92d2](https://github.com/aigentive/ralphx.app/commit/055e92d27a48db60138ca425e940c24d6ac12c41)

## Fixes And Polish
- Daily release automation now skips maintenance-only ranges when every commit after the latest release tag includes a release-skip marker such as `[skip daily-release]`, `[skip release]`, `[no daily-release]`, or `[no release]`. [055e92d2](https://github.com/aigentive/ralphx.app/commit/055e92d27a48db60138ca425e940c24d6ac12c41)
- Release preparation now keeps `src-tauri/Cargo.lock` in the staged version-bump path, with the root `ralphx` package lockfile version synced for the release flow. [8d569e24](https://github.com/aigentive/ralphx.app/commit/8d569e240907c792a95665c2b993fc984eacbb77)
