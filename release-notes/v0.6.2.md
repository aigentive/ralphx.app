# RalphX.app v0.6.2

RalphX.app 0.6.2 focuses on forked-PR coverage gating and clearer contributor CI behavior.

## User-Facing Changes
- No app runtime, UI, installation, or end-user workflow changes are included in this release. [97dc80d5](https://github.com/aigentive/ralphx.app/commit/97dc80d531ec6c403629e8405f756a1cd7c785cd)

## Developer And Maintainer Changes
- Coverage CI now handles fork PRs where GitHub withholds repository secrets: local coverage jobs remain blocking, while the external Codecov patch-status wait is skipped after prerequisites pass. [97dc80d5](https://github.com/aigentive/ralphx.app/commit/97dc80d531ec6c403629e8405f756a1cd7c785cd)
- Internal coverage-required PRs now fail quickly and explicitly when `CODECOV_TOKEN` is unavailable, distinguishing misconfiguration from expected fork behavior. [97dc80d5](https://github.com/aigentive/ralphx.app/commit/97dc80d531ec6c403629e8405f756a1cd7c785cd)

## Known Issues
- Fork PRs do not receive Codecov’s 90% patch gate because secret-backed upload is unavailable for those runs. [97dc80d5](https://github.com/aigentive/ralphx.app/commit/97dc80d531ec6c403629e8405f756a1cd7c785cd)
