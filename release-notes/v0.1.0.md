# RalphX v0.1.0

This initial 0.1.0 release focuses on Claude/Codex multi-harness execution, first-class delegation and verification, and provider-aware chat attribution and usage reporting.

## Highlights
- Claude and Codex sessions now keep their selected harness through bootstrap, recovery, relaunch, and pause/resume; chat headers render provider context cleanly, and unavailable ideation harnesses now fail explicitly. (`42b26250`)
- Chat now surfaces attribution and usage across the product: RalphX persists message and run attribution, shows scoped usage aggregates in the header stats popover, task cards, and insights, and hydrates chat stats during live turns. (`42b26250`)
- Delegated work now renders as task cards with lineage and visual status; cards expand into child transcripts, replay from active state, and refresh on child events instead of falling back to raw delegate tool calls. (`42b26250`)
- Verification now runs on backend-owned rounds and native snapshots; verifier child sessions launch without blocking the parent workflow by default, chat shows verification delegate progress, and round widgets plus rerun history read from authoritative snapshot state. (`42b26250`)
- Settings now reorganize planning, repository, and review controls: Git and GitHub are merged under Repository, Ideation becomes Planning & Verification, review policy gets a dedicated section, lane model/effort selectors show presets and effective values, and merge validation defaults to Disabled. (`42b26250`)

## Fixes And Polish
- Codex activity now surfaces directly in chat: command execution and file-change activity are visible, and file edits render as diffs instead of opaque change notices. (`42b26250`)
- Chat chrome is cleaner: session headers were decluttered, status moved under the header context, empty ideation header chrome is hidden, and external ideation sessions get default titles. (`42b26250`)
- Live and replayed chat state is more reliable: Codex stats and widgets hydrate correctly again, streamed Claude content blocks persist for replay, provider-role rendering is locked down, and duplicate Codex tool-call lifecycle snapshots are deduped. (`42b26250`)
- Ideation startup is steadier: bootstrap gates correctly, fresh launches keep the right mode, session naming is stabilized, and progress stays scoped to the current plan. (`42b26250`)
- Merge and retry workflows are steadier: manual task retries restart execution, freshness-return completions are classified separately, reaccepted plans keep the right base branch, and archived tasks stay out of the merge pipeline. (`42b26250`)
